# Server Admin Guide

> ⚠️ **重要提示：** 本指南涵盖普通（无 Mod）专用服务器的搭建。
> 如果你要搭建带有 Mod 的服务器，请参考我们更新的 **Mod 服务器指南** [点此查看](https://mod.io/g/insurgencysandstorm/r/how-to-set-up-a-steam-dedicated-server-with-mods)


## 安装（Installation）

### 前置条件 [仅限 Windows]

在继续安装服务器之前，请确保已安装以下前置组件。

- [Visual C++ 2015 Redistributable](https://www.microsoft.com/en-us/download/details.aspx?id=53587)
- [Visual C++ 2017 Redistributable](https://aka.ms/vs/15/release/vc_redist.x64.exe)

### SteamCMD

与其他在 Steam 上发布的游戏一样，Insurgency: Sandstorm 的服务器文件通过一个名为 `SteamCMD` 的命令行工具发布。这个无图形界面的 Steam 客户端可以让你轻松安装和更新服务器文件。

你可以从以下链接下载 SteamCMD：

- [Windows](https://steamcdn-a.akamaihd.net/client/installer/steamcmd.zip)
- [Linux](https://developer.valvesoftware.com/wiki/SteamCMD#Linux)

如果你使用的是 **Windows**，请将 **SteamCMD** 解压到一个已知文件夹位置，或创建一个新文件夹用于安装。在安装过程中，你会频繁地回到此位置。

通过 SteamCMD 执行的大多数命令都需要知道游戏的 App ID。Insurgency: Sandstorm 的 App ID 是 `581330`。

要进行基本的服务器设置，请按照以下说明操作：

1) 打开 `SteamCMD`，等待其更新。这可能需要几分钟时间。
2) 输入 `login anonymous` 并按回车。这会以匿名方式登录，允许你下载大多数游戏服务器。
3) 输入 `app_update 581330 validate` 并按回车。这将安装并验证 Insurgency: Sandstorm 专用服务器。
4) 一旦收到提示应用已完全安装的消息，输入 `quit` 并按回车。

现在你可以在 Steamapps 文件夹内的 common 子文件夹中找到你的专用服务器。完整路径应类似：**SteamCMD\\steamapps\\common**。你可以创建此文件夹的快捷方式以便日后访问。

请注意，SteamCMD 的根目录会因你为其命名的名称而异。

你可以**在[此处](https://developer.valvesoftware.com/)**找到有关 SteamCMD 及其使用方式的更多信息。

### 启动（Launching）

安装 SteamCMD 后，你需要创建一个脚本以使用你想要的选项和设置来启动服务器。在 Windows 上，这是一个 **批处理（batch）** 文件；在 Linux 上，这是一个 **Shell** 脚本。

**对于 Windows**：

1) 在 `sandstorm_server` 目录中创建一个新的文本文档。
2) 打开它，按下 `CTRL + Shift + S`。会弹出一个窗口要求你保存文件。
3) 将文件命名为 `start.bat`，然后按回车。

**对于 Linux**：

1) 运行 `cd .steam/steamapps/common/sandstorm_server` 导航到新创建的服务器目录。
2) 运行 `touch start.sh` 并按回车。

将在指定位置创建一个指定名称的新文件。你需要使用适用于你操作系统的特定命令行来编辑此文件以启动服务器。

命令行应包含起始地图、场景以及任何其他启动参数。例如：

| 平台 | 命令行 |
|---|---|
| Windows | `InsurgencyServer.exe Oilfield?Scenario=Scenario_Refinery_Push_Security?MaxPlayers=28 -Port=27102 -QueryPort=27131 -log -hostname="My Server"` |
| Linux | `Insurgency/Binaries/Linux/InsurgencyServer-Linux-Shipping Oilfield?Scenario=Scenario_Refinery_Push_Security?MaxPlayers=28 -Port=27102 -QueryPort=27131 -log -hostname="My Server"` |

在此示例中，服务器将在 Refinery（Oilfield）的 Push Security 模式下启动，托管在端口 27102 上，最大玩家数为 28。如果地图或场景名称存在拼写错误，服务器将默认使用 Farmhouse。如果发生这种情况，请验证你在命令行中拼写的一切是否正确。

### 端口转发（Port Forwarding）

为了让你的服务器在服务器浏览器中显示，你需要同时转发 `TCP` 和 `UDP`。你可以在很多网站上找到端口转发的教程。我们推荐 [portforward.com](https://portforward.com/)，该网站提供了大多数路由器制造商的教程。

你需要将已转发的端口放入之前创建的脚本中。例如，如果你转发了端口 `27102` 和 `27131`，你的命令行应如下所示：

| 平台 | 命令行 |
|---|---|
| Windows | InsurgencyServer.exe Oilfield?Scenario=Scenario_Refinery_Push_Security?MaxPlayers=28 `-Port=27102` `-QueryPort=27131` -log -hostname="My Server" |
| Linux | Insurgency/Binaries/Linux/InsurgencyServer-Linux-Shipping Oilfield?Scenario=Scenario_Refinery_Push_Security?MaxPlayers=28 `-Port=27102` `-QueryPort=27131` -log -hostname="My Server" |

### 防火墙（Firewall）

现在你已经转发了端口，接下来需要确保防火墙允许连接，以便你的服务器出现在服务器浏览器中。

**对于 Windows**：

1) 按下 `Windows + R`。这将打开"运行"窗口。
2) 粘贴以下字符串：`wf.msc` 并按回车。
3) 点击左侧面板的 `入站规则（Inbound Rules）`，然后点击右侧面板的 `新建规则（New Rule）`。
4) 选择 `端口（Port）` 并点击下一步。
5) 选择 `TCP`，确保已选择 `特定本地端口（Specific local ports）`，然后将你转发的端口以逗号分隔填入框中（例如，我们的游戏端口是 `27102`，查询端口是 `27131`）。
6) 选择 `允许连接（Allow the connection）` 并点击下一步。
7) 确保勾选所有复选框，然后点击下一步。
8) 将规则命名为：`Insurgency Server` 并点击完成。

重复步骤 3 到 8，但在规则 5 中，你需要允许 **UDP** 而不是 **TCP**。最终，你应该在入站规则部分有两个新规则。

你可以通过回到入站规则选项卡，右键点击你创建的规则，然后点击属性来编辑你的规则。

**对于 Linux**：

建议参考你所用发行版的文档来配置防火墙。

将**入站规则**添加到防火墙后，你现在可以使用之前在**启动**部分创建的批处理/shell 脚本文件启动服务器了。请注意，服务器将以默认配置运行。默认配置会循环遍历所有可用的 **Versus PVP** 场景，并遵循**官方游戏规则**。

如果你希望修改服务器的配置，可以在本指南的以下部分中找到所需信息。

## 配置（Configuration）

### 命令行参数（Command Line Arguments）

大多数服务器配置通过 .INI 文件和启动参数完成。这些文件可以使用任何文本编辑器进行编辑，如记事本（Notepad）、Notepad++、Sublime Text 和 VSCode。

配置文件可以在以下目录中找到（相对于服务器的安装目录）：

**对于 Windows**：`Insurgency\\Saved\\Config\\WindowsServer`
**对于 Linux**：`Insurgency/Saved/Config/LinuxServer`

如果目录尚不存在，你需要手动创建一个。你还需要创建新的 .INI 文件：

**对于 Windows**：

1) 在 Config 目录（路径：`Insurgency\\Saved\\Config`）中创建一个名为 `WindowsServer` 的新文件夹。
2) 在 `WindowsServer` 文件夹中创建一个新的文本文档。
3) 打开它，按下 `CTRL + Shift + S`。会弹出一个窗口要求你保存文件。
4) 将文件命名为 `Engine.ini` 并按回车。
5) 再次打开新的文本文档，按下 `CTRL + Shift + S`。
6) 将文件命名为 `Game.ini` 并按回车。

**对于 Linux**：

1) 输入 `mkdir -p Insurgency/Saved/Config/LinuxServer`。如果缺少该目录，这将创建一个新的 LinuxServer 目录。
2) 输入 `cd Insurgency/Saved/Config/LinuxServer` 导航到新创建的 LinuxServer 目录。
3) 输入 `touch Engine.ini` 并按回车。
4) 输入 `touch Game.ini` 并按回车。

两个具有指定名称的新 INI 文件将在该位置创建。

| 参数 | 说明 |
|---|---|
| `-log` | 启用日志窗口。 |
| `-hostname="My Server"` | 设置服务器在服务器浏览器中显示的名称（将 "My Server" 替换为你选择的名称）。 |
| `-EnableCheats` | 启用作弊功能以进行测试。 |
| `-Port=xxxx` | 游戏连接的 UDP 端口（将 XXXX 替换为端口号）。 |
| `-QueryPort=xxxx` | Steam 运行服务器查询的端口（将 XXXX 替换为端口号）。 |

### 旅行参数（Travel Parameters）

旅行参数放在命令行中地图名称的末尾，后跟问号（?），然后是设置变量及其值。在上面的示例中，MaxPlayers 就是一个旅行参数。

| 参数 | 说明 |
|---|---|
| `Password` | 为服务器设置密码。 |
| `MaxPlayers` | 设置可以加入服务器的最大玩家数。在合作服务器上，这是人类玩家的数量。 |

&nbsp;

### 管理员列表（Admin List）

游戏内置了一个管理员菜单，提供基本功能，如踢人、封禁和切换关卡。要添加管理员，你需要知道他们的 64 位 Steam ID（也称为 steamID64）。你可以使用在线转换器（如 STEAMID I/O）来查找。

然后，你需要在以下目录中创建一个名为 `Admins.txt` 的文件：**Insurgency/Config/Server**。

**对于 Windows**：

1) 在根目录的 Insurgency 文件夹中创建一个名为 `Config` 的新文件夹。
2) 在新创建的 `Config` 文件夹中创建一个名为 `server` 的新文件夹。你的路径现在应类似于：**Insurgency/Config/Server**。
3) 在 `server` 文件夹中创建一个名为 `Admins.txt` 的新文本文档。

**对于 Linux**：

1) 运行 `mkdir -p Insurgency/Config/Server`。这将创建新的 Server 目录。
2) 运行 `cd Insurgency/Config/Server` 导航到新创建的 Server 目录。
3) 运行 `touch Admins.txt`

你也可以通过指定命令行开关来覆盖该文件。例如，**-AdminList=OtherAdminList** 会使服务器改为读取 **OtherAdminList.txt**。

对于每个要添加的管理员，你需要将其 **steamID64** 放在 **Admins.txt** 文件的新一行中。你可以为任意数量的管理员进行此操作。添加新管理员后，需要重启服务器或更换关卡，所做的任何更改才能在游戏中生效。当具有管理员权限的玩家连接到服务器时，他们将能够打开管理员菜单，默认绑定到小键盘减号键，并可以在按键绑定菜单中重新绑定。

### 每日消息（Message of the Day）

可以配置服务器使用 MotD 文件在加载画面期间播报每日消息。默认情况下，此文件必须放置在 `Insurgency/Config/Server/Motd.txt`。这可以使用 `-motd` 参数在命令行上定义。如果你希望它读取 `MyOtherMOTD.txt`，则需要指定 `-motd=MyOtherMOTD`。

如果此文件不存在，你需要创建它。

**对于 Windows**：

1. 在 Server 文件夹中创建一个名为 `Motd.txt` 的新文本文档。

**对于 Linux**：

1. 运行 `cd Insurgency/Config/Server` 导航到 Server 目录。
2. 运行 `touch Motd.txt` 创建文本文件。

该文件不允许任何格式设置，只会显示纯文本。

### 远程访问 / Rcon（Remote Access）

可以启用 Rcon 为服务器提供远程访问，用于不需要你加入服务器的管理任务。这是使用 [Source Engine Rcon Protocol](https://developer.valvesoftware.com/wiki/Source_RCON_Protocol) 实现的，已有多款客户端可用，例如 [mcrcon](https://sourceforge.net/projects/mcrcon/) 和 [Simple SourceDS RCON](https://sourceforge.net/projects/ssrcdsrcon/)。[RCON Wrapper 设置和已知问题](https://mod.io/g/insurgencysandstorm/r/rcon)。

### 启用访问（Enabling Access）

可以通过修改 **Game.ini** 文件或通过服务器的**命令行**来启用 Rcon。要通过命令行启用，可以提供以下参数：

`-Rcon -RconPassword=password -RconListenPort=27015`

密码和端口参数的值可以替换为你偏好的值。

通过 Game.ini，你可以添加以下选项：

```
[Rcon]
bEnabled=True
Password=password
ListenPort=27015
```

以下额外的配置变量也可用：

| 变量 | 默认值 | 说明 |
|---|---|---|
| `bUseBroadcastAddress` | True | 如果启用，rcon 套接字将监听所有可用的网络设备。 |
| `ListenAddressOverride` | 0.0.0.0 | 如果 bUseBroadcastAddress 为 False，这是 rcon 套接字绑定的网络 IP。 |
| `bAllowConsoleCommands` | True | 启用后，任何未知的 rcon 命令将被解释为控制台命令。 |
| `MaxPasswordAttempts` | 3 | 在 IP 地址被临时封禁之前允许的最大失败认证尝试次数。 |
| `IncorrectPasswordBanTime` | 30 | 客户端在达到最大尝试次数后被禁止尝试连接 rcon 的时间（以分钟为单位）。 |

### 命令（Commands）

你可以随时通过在控制台中输入 `help` 来获取命令列表。这将显示所有可用命令的摘要、它们的用法以及命令功能的描述。

下表显示了可用的命令。必选参数用"**<**"和"**>**"括起来，可选参数用"["和"]"括起来。"net ID"通常指用户的 Steam ID。

| 命令 | 参数 | 说明 |
|---|---|---|
| `help` | | 显示命令列表。 |
| `listplayers` | | 列出当前连接到服务器的玩家。 |
| `kick` | <id/netid/name> [reason] | 将玩家从服务器踢出。 |
| `ban` | <id/netid/name> [duration in minutes] [reason] | 将玩家从服务器封禁。 |
| `banid` | [duration in minutes] [reason] | 通过 net ID 封禁服务器玩家。不需要玩家在服务器上。 |
| `permban` | <id/netid/name> [reason] | 永久封禁服务器玩家。 |
| `unban` | | 解除对某用户的封禁。 |
| `listbans` | | 显示服务器的封禁列表。 |
| `say` | | 在聊天框中向所有玩家显示一条消息。 |
| `restartround` | [0 = no team switch, 1 = swap teams] | 重新开始当前回合。 |
| `maps` | [level filter] | 列出可用地图。 |
| `scenarios` | [level filter] | 列出可用场景。 |
| `travelscenario` | | 将关卡更改为给定场景。 |
| `travel` | | 将服务器转换到不同的关卡。 |
| `gamemodeproperty` | [new value] | 在当前场景长度内获取或设置游戏模式属性。 |
| `listgamemodeproperties` | [property filter] | 列出当前加载的游戏模式的所有可用属性。 |

默认情况下，任何未知命令将作为控制台命令执行。

## 启用统计（Enabling Stats）

任何 Insurgency: Sandstorm 社区服务器都可以托管启用统计功能的游戏。在服务器上启用统计功能同时也会启用在服务器上游玩时的经验值获取。唯一的要求是你必须使用 Steam GSLT，并且不能设置服务器密码。

**1.** 访问 [GameStats Token Generator](https://gamestats.sandstorm.game/) 并通过你的 Steam 个人资料进行认证。

![mceclip0.png]( https://image.modcdn.io/articles/812b/95/auto/mceclip0.png)

![mceclip0.png]( https://image.modcdn.io/articles/812b/95/auto/58488mceclip0.png)

**2.** 点击"Generate Token"，这将为你创建一个 **GameStats** 令牌。

![mceclip4.png]( https://image.modcdn.io/articles/812b/95/auto/mceclip4.png)

你的令牌看起来类似如下：

`-GameStatsToken=1417264D1C6549CC95E10CA1E9BE8F09`

只需将生成的令牌添加到你的启动参数中，统计功能就会启用！请确保**不要与第三方分享你的 GameStats 令牌**。被发现滥用系统的服务器管理员将被撤销令牌，你的账户也将被调查。如果你怀疑你的 GameStats 令牌已被分享，请生成一个新的令牌。

### GSLT 认证（GSLT Authentication）

为了让你的服务器向统计服务器进行认证，你必须通过 **Steam GSLT**（Game Server Login Token，游戏服务器登录令牌）认证你的服务器。GSLT 可以从 Steam 的**[游戏服务器账户管理](https://steamcommunity.com/dev/managegameservers)**页面获取。正如该页面所述，你的 Steam 账户需要满足各种标准才能生成令牌。这包括**拥有该游戏**、**将你的 Steam 账户与手机号码绑定**以及**账户信誉良好**。

要为 Insurgency: Sandstorm 生成令牌，请输入 App ID：**581320**。备注字段可以设置为任何内容，用它来帮助你记住分配给每个服务器的令牌。你托管的每个服务器都需要一个唯一的令牌。

要使用 GSLT 认证启动服务器，请在服务器的命令行中添加 `-GSLTToken=xxxxxxxxxxxxxxx`，其中 xxxxxxxxxxxxxxx 是 Steam 游戏服务器账户页面上显示的令牌。

### 统计功能选择加入（Stats Opt-in）

默认情况下，游戏服务器不会尝试向统计系统报告。按照上述 GSLT 登录步骤，在服务器命令行中添加 `-GameStats` 将启用到该系统的选择加入。

### 可选：官方规则（Optional: Official Rules）

游戏附带了一套被认为是**官方规则**的规则集；这会锁定你在专用服务器上可以更改的选项。如果你运行的是一个基本原版的服务器，你可能需要考虑选择加入此规则集，使你的服务器出现在"**官方规则**"过滤条件下。要选择加入，请在服务器命令行中添加 `-ruleset=OfficialRules`。

## 地图轮换（Map Cycle）

默认情况下，服务器会循环遍历所有可用的 Versus PVP 场景。

可以使用包含一系列场景（以换行符分隔）的地图轮换文件配置服务器。默认情况下，此文件必须放置在 **Insurgency/Config/Server/MapCycle.txt**。这可以使用 `-MapCycle` 参数在命令行上定义。因此，如果你希望它读取 **MyOtherMapCycle.txt**，则需要指定 `-MapCycle=MyOtherMapCycle.txt`。

如果此文件不存在，你需要创建它。

**对于 Windows**：

1) 在 Server 文件夹中创建一个名为 `MapCycle.txt` 的新文本文档。

**对于 Linux**：

1) 运行 `cd Insurgency/Config/Server` 导航到 Server 目录。
2) 运行 `touch MapCycle.txt` 创建文本文件。

例如，一个仅限 Skirmish 模式的服务器的 MapCycle.txt 应如下所示：

```
Scenario_Crossing_Skirmish
Scenario_Hideout_Skirmish
Scenario_Precinct_Skirmish
Scenario_Refinery_Skirmish
Scenario_Farmhouse_Skirmish
Scenario_Summit_Skirmish
```

此外，你可以定义一个带有覆盖游戏模式的条目，因为某些游戏模式可以共享场景。例如，要在 Hideout 上进行 Hardcore Checkpoint Security，你需要输入以下内容：

`(Scenario="Scenario_Town_Checkpoint_Security",Mode="CheckpointHardcore")`

### 光照场景（Lighting Scenarios）

除了白天场景外，还提供夜间场景。这通过旅行参数 `?Lighting=` 控制，有效值为 **Night** 或 **Day**。

对于地图轮换，你通过 Lighting 参数来控制，类似于 Mode 参数。你应确保指定此参数以显示正确的投票图片。例如：

**(Scenario="Scenario_Town_Checkpoint_Security",Lighting="Night")**

这将设置地图轮换条目为使用 Hideout（Town）的夜间 Checkpoint Security 场景。你可以通过显式定义 Day 和 Night 的 Lighting 参数，在同一个地图轮换中混合 Day 和 Night 场景，例如：

**(Scenario="Scenario_Town_Checkpoint_Security",Lighting="Night") (Scenario="Scenario_Town_Checkpoint_Security",Lighting="Day")**

这将允许你在 Day 和 Night 两种模式下玩 Hideout Checkpoint Security，并在投票屏幕上显示相应的图片。

## 玩家投票（Player Voting）

社区服务器可以选择在其服务器上启用玩家投票。可以通过在 Game.ini 中输入以下内容来启用：

**[/Script/Insurgency.TeamInfo]**

**bVotingEnabled=True**

**TeamVoteIssues=/Script/Insurgency.VoteIssueKick**

这将启用投票，并包含踢人投票议题，这是目前唯一受支持的投票议题。当有额外的投票议题可用时，你需要为每个要启用的议题在下方添加新行。

启用投票后，玩家可以按 Esc 并点击"Call Vote"来发起某个投票议题的投票。按 F1 投赞成票，按 F2 投反对票。
